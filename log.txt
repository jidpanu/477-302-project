--- อธิบายขั้นตอนการใช้ git จาก log นี้ ---

1. สร้าง branch ใหม่ชื่อ dev-jidpanu เพื่อแยกงานจาก master  
   คำสั่ง: `git branch dev-jidpanu`

2. สลับไปทำงานบน branch dev-jidpanu  
   คำสั่ง: `git switch dev-jidpanu`

3. แก้ไขไฟล์ index.html แล้วเช็คสถานะด้วย git status  
   คำสั่ง: `git status`

4. ดูความแตกต่างของไฟล์ที่แก้ไขกับไฟล์เดิม  
   คำสั่ง: `git diff`  
   (แสดงรายละเอียดว่าแก้ไขอะไรไปบ้างในไฟล์)

5. ใช้ git add . เพื่อเพิ่มไฟล์ที่แก้ไขเข้า staging area  
   คำสั่ง: `git add .`

6. ใช้ git commit -m "add menu" เพื่อบันทึกการเปลี่ยนแปลงพร้อมข้อความ  
   คำสั่ง: `git commit -m "add menu"`

7. ใช้ git push --set-upstream origin dev-jidpanu เพื่อส่ง branch dev-jidpanu ขึ้น GitHub และตั้งให้เชื่อมกับ remote branch  
   คำสั่ง: `git push --set-upstream origin dev-jidpanu`

8. สลับกลับไป branch master ด้วย git switch master  
   คำสั่ง: `git switch master`

9. ใช้ git merge dev-jidpanu เพื่อรวมงานจาก dev-jidpanu เข้าสู่ master (fast-forward)  
   คำสั่ง: `git merge dev-jidpanu`

10. ใช้ git push เพื่อส่งการเปลี่ยนแปลงของ master ขึ้น GitHub  
    คำสั่ง: `git push`

**สรุป:**  
เป็น workflow พื้นฐานสำหรับการพัฒนาโปรเจกต์แบบทีมด้วย git โดยแยกงานเป็น branch, commit, push, merge และ sync กับ remote
